# Repository Map

## Execution flow

1. The parser in [src/thesis_elm/cli.py](../src/thesis_elm/cli.py) defines the subcommands and shared arguments.
2. Each subcommand dispatches into a handler in [src/thesis_elm/experiments.py](../src/thesis_elm/experiments.py).
3. Dataset loading, train/test splitting, standardization, and synthetic shifts live in [src/thesis_elm/data.py](../src/thesis_elm/data.py).
4. Model instances are constructed by `create_model(...)` and share a common `BaseClassifier` API.
5. Metrics are timed, flattened into CSV rows, and written through [src/thesis_elm/utils.py](../src/thesis_elm/utils.py).

## Core directories

- `../src/thesis_elm/`: implementation code
- `../src/thesis_elm/models/`: four model families plus shared base classes
- `../docs/`: thesis-oriented prose and experiment guidance
- `../tests/`: smoke tests for CLI, models, and result rows
- `../results/`: CSV outputs generated by experiment commands

## Main abstractions

- `DatasetBundle`: transport object for train/test splits and metadata
- `BaseClassifier`: common `fit`, `predict`, and `score` interface
- `GradientClassifier`: shared training loop for Logistic Regression and MLP
- `ELMClassifier`: frozen hidden layer plus closed-form output solve
- `OSELMClassifier`: online updates through recursive least squares

## Research workflow mapping

- Baselines: `run`
- Hidden-neuron sweep: `grid-search`
- Scalability and learning curves: `scaling`
- Continual learning and covariate shift: `sequential`
